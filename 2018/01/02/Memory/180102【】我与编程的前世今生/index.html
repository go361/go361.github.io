编程学习之路

##自由生长:N次擦肩
###程序 = 流程图
最早接触电脑是在高中时期(2001~2004)，看了微机课发的教材，我对程序的理解是“程序=流程图”，我曾经天真的以为只要画出下面流程图中神奇的菱形，它就会自动选择不同路径去实现对应的功能。

###程序 = Turbo C + CMD
大学第一学期(2004下半年)“计算机文化基础”课上接触到1.44MB软盘这种神奇的存储介质，可以把课堂上自己创建文档存储在里面，下次课继续使用。 (//:此时我仍做着在word中画一个神奇的菱形去实现相应功能的白日梦)


懵懂的我直到第二学期(2005上半年)“高级语言程序设计”课时，才明白“程序=流程图”的理解是多么荒谬，当时上机课操练C语言的环境是“Turbo C + CMD”。经过一学期课程学习，05年9月侥幸通过国家计算机等级考试（二级C语言，上机时我只做了60分的题)。同学习大多数课程的情形一样，考试后我的编程学习也告一段落。

C语言之外:汇编、Matlab

大三下(2007上半年)学习“单片机原理及应用”和“Matlab语言及应用”两门课程时，我才知道编程语言不仅仅是C这一种。 2007年春节期间，在“微机原理及接口技术”课程基础上，我自学了单片机相关的书籍，单片机课程的学习不算太吃力；Matlab课程则是完全开辟了新的战场，以矩阵作为核心数据结构的这种语言颠覆了我对于编程的认知。
临近毕业时用Matlab实现毕业设计中多相滤波算法的仿真，算是我第一次相对高端的编程体验。
###界面并不难:Delphi
再次接触编程已是本科毕业两年之后(2010年下半年)，XTY培训班开设的“Delphi程序设计”课程中，老师一步步带领大家设计界面、编写事件响应函数，实现简版的计算器(模仿windows计算器)。最终课程大作业，我提交了自己编写的扫雷程序(模仿Windows扫雷)。
此外培训期间受大学同班的严同学的启发自学《计算机网络》、《C语言程序设计-现代方法》、《数据结构-C语言版》书籍，然而当时并未购买笔记本电脑，学习只停留在书本理论层面，没动手实践。
###程序 = 数据结构 + 算法
2011年9月开始，断续自学孙鑫《C++视频课程》至第11讲，2011年底在网上看到如下迷宫题目:

从左边入口处的 2011 进去，在迷宫里转悠，最后变成 2012 从右边出来。你可以在迷宫里转圈，可以重复之前走过的路，但不能往回退着走。你能成功走出来吗？

尝试编程求解这个迷宫题的过程中，我首次认识到数据结构的重要性!
###读研 : DSP编程、MFC
2016年3月，研究生毕业论文是硬件相关的题目，控制芯片选用TMS320F2833xDSP，开发环境CCS支持C语言编程。在没有任何硬件编程基础的情况下，依靠宿舍学霸ZJ的帮助从头开始实现了按键控制(包含去抖动)、数码扫描显示浮点数、占空比可控软启动、ADC采样等功能。
受舍友ZJ的影响重燃了学习C++的激情，曾自学吕鑫的《MFC视频教程》至第五章……!
###Python:机器学习
研究生毕业回到工作岗位5个月之后，时间进入2017年10月，受LX同学的鼓励开始学习Python这门因为人工智能逐渐火起来的编程语言，已累计投入约100小时。
##任务驱动:愿我能
俗话说“常立志，不如立长志”，回顾十多年来自己在编程学习上曾发起过数次行动，每一次都是浅尝辄止、无疾而终，均未能投入更多时间和精力去持续学习。究其原因主要在于学习没有目标，处于一种自由生长的散漫状态。如今开启学习Python的新征程，当设立合理的目标和计划，转入任务驱动模式，扎扎实实地行动，持续学习3年以上。
