Python小组2018行动记录

2017年11月开始自学Python，18年恰好加入了一个学习小组。这篇文章是学习Python的一个阶段性总结。

PartI : 行动概况（2018年200h）

以下分为几个部分回顾我Python方面的理论学习和编程实践，其中第一部分为2017年，其余为18年加入小组后的行动记录。

加入小组前的自学（2017年88h）

2017年11月20日开始自学 Python，加入Python小组之前累计学习时长88h，包括以下内容:

  • 《父与子的编程之旅》约32小时
  • 《核心编程》（第2版）11.19-12.5
  • “网易云课堂” 约17小时：Python入门课(by老男孩)
    《父与子的编程之旅》通俗易懂，通过一个个简单的小程序讲解Python基础知识，实现了一些简单的GUI界面，运用Pygame模块实现了反弹球和滑雪者游戏，最终给出了Python Battle下的吃豆人游戏，并鼓励读者写出能打败CircleAI的机器人(要实现最佳策略似乎比较难)。这本书非常适合自学，小学高年级或者初中生就可以很容易上手学习。
    《核心编程》是一本权威著作，讲解数据类型时给出了所有的工厂函数和内置函数，更适合当做工具书来查找一些知识点。前言中作者是这样说的“我们将目标锁定于那些已经熟悉其他高级语言的人士和大、中学生。” 有其他语言编程经验的人快速迁移至Python时，这本书应该是不错的选择。
    网易云课堂的入门课程整体来说知识密度偏低，经过该课程介绍开始使用PyCharm，该IDE环境下”Ctrl+鼠标左键点击“即可快速跳转至相应类型或函数的源代码处，使用起来极其方便，另外批量修改变量名的功能也非常实用!

PartII 分析与思考

目标不明确

写这篇总结时，重新阅读了以下四篇文字的内容：

  • 【P002】知行合一，从点滴做起180101（年初制定的目标）
  • S53C成长贴1800
  • 深度学习小组的申请书
  • Python小组的申请书
    回顾一年以来的行动，自己存在的最大问题就是“想法太多，目标不明，行动不足”。这两三年来S的文章没少读，成长课没少听，但是实际行动中仍然没有做到“先把一件事情做好，持续行动，力保稳定后再逐渐挂载第二件事”。
    成长进步是要突破自己的舒适区的，能在一个方面做好已实属不易。同时加入两个成长类小组（还是不能换脑子的俩小组），两个目标撕扯自己有限的时间和精力，最终两个方向都没有什么大的进步。

对生活的预估不足

年初时，妻子和孩子们还在洛阳同爸妈一起生活，夫妻两地分居，我工作上也恰逢相对空闲的时段，周末时一个人的生活自由度也相当大（每天可以有45个小时的时间用于学习）。在这段时间内制定计划无疑使得自己的欲望膨胀得十分厉害。2月中旬开始休假进入春节模式时，一切都乱了套，基本上各个小组的行动几乎都跟不上节奏了。3月6月为相对平稳期，6月23日至9月16日累计用时104.5小时备考一级建造师，小组行动量也近乎中断。20180910—20180930到重庆出差学习，后半段一下课就疯狂地放纵，一周内看了15个小时的羽毛球比赛视频。十一之后工作中还穿插着半年竞赛性考核、工程师评审及年终考核等。这些都远远超出了年初时的预估，致使一年来行动节奏时好时坏，极不稳定。
年初计划时总是豪情万丈，觉得一年的时间挺长的，似乎可以做好多想事情。但这一整年是由一天一天累加起来，实际上每一天的生活要照顾到工作和家庭，能专心做事情的时间十分有限，很难做到“每天行动2小时，全年不间断”。一年以来，共计行动不足250小时，距离自己申请书中的每周10小时的目标相去甚远。

行动量不够，写代码太少

编程是一门手艺，要想达到熟练的境界离不开大量的练习，只有通过大量的练习、写足够数量的代码才会慢慢养成习惯，相关的规则和用法烂熟于心，才可能逐渐的培养出编程的感觉以及解决问题的思路。
一年中用于实际编程练习的时间约为120小时，在投入的总时间中占比不足一半。因为写代码的数量少，理论部分的学习没有及时应用于编程练习中，所以理论学习的效果也大打折扣。
整体来说学习没有按照组长的设计思路走，东一榔头西一棒槌，学东西几乎都停留在吸收资讯的低层次水平，没有很好地应用起来。每一阶段的行动都存在较严重的放水行为。

PartIII 2019年展望

考博？

18年10月查看系统内学校的博士招生目录，选定了一个与工作联系比较紧密的方向，根据考试科目购买了一些专业书籍。11月份开始逐步学习《离散数学》，《计算机操作系统教程》……
客观地说，对我来说考博士的难度不小。首先英语方面的问题最大，学了这么多年的英语，硕士毕业了还没有通过CET6的考试，说来真是惭愧。其次数学考试我计划选择《离散数学》，目前学习进度卡在第二章“谓词逻辑”尝试了两三次都没能理解通畅；后面章节的内容也几乎是全新的知识，也有一定难度。专业课考试计划选“软件基础”，共包含《C语言程序设计》、《编译原理》、《计算机操作系统教程》三门课程，C语言只有指针之前的部分熟悉一些，后面的需要重新学习；《编译原理》是之前完全没有接触过的课程，应该是难度最大的一门；《操作系统》的书过了第一遍，虽然理解上无大碍，但是知识点好像一个也没记住。

对小组的期待

如果还有小组，还是null组长来带的话。

  1. 之前好像听组长提到过考学的想法，希望我学习过程中遇到问题时，能得到组长及其他成员的耐心指点。
  2. 学习C语言我计划刷的书是《C语言程序设计：现代方法》，不知道是否能同小组的行动结合起来，能有一起学习的小伙伴就更好啦。
  3. 相对比较大的编程作业还是比较渴望组长视频直播式的分享，比如说“阿皮”同学实现的week78，我自己看起来十分吃力，可能花了67个小时，都没有理清他的框架设计思路。
  4. 感觉上分享到群文件中的资料很容易变得混乱，可能用“坚果云”的网盘共享一下会比较好，我愿意担任学习资料的整理工作。进度表的excel也可以实现多人共享的编辑，免去下载上传的麻烦。
