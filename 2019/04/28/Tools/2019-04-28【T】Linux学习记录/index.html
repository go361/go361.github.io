Linux学习记录

Linux再经典、再强大，充其量也只是个操作系统，是一个工具。虽说“磨刀不误砍柴工”，但是一定要对自己所要砍的“柴”有清醒的认识，不能盲目地在工具上投入过多的经历，在耽于磨刀的过程中失去方向，还用所谓的“不误”来为自己的踌躇不前辩解。

##1、初识Linux
2008年6月临近大学毕业的时候，我有幸收到一位同学的推荐邀请，注册加入hdchina，接触到PT的文件分享机制，下载收藏了一批高清析度的视频和音乐。《Revolution.OS.2001.DVDRip.XviD-RETRO.avi》是其中的一个，影片内容是关于计算机操作系统中Windows和Linux两大阵营之间的竞争。通过观看这部影片，我了解到Linux是一种完全不同于Windows的操作系统——Linux自创立之初就是开放源代码源、支持自由软件的。
第一次成功安装Linux操作系统已经是2011年2月时的事情了。当时在单位同事的建议下将ASUS笔记本操作系统更换为Win7（之前一直用WindowsXP），整理完台式机的硬盘数据后，台式机空闲出来，在上面尝试安装了Ubuntu操作系统，看过几个教学视频后，没有更深入地去学习。之后的七年中除了使用安卓手机外，与Linux无任何交集。
（没有找到好的教程和学习材料或许是没有继续学习Linux的原因之一。单位周边的第一个3G基站是2011年2~3月份才建立起来的，更早些的时候工作单位不仅没有互联网，也没有手机3G网络信号，几个玩魔兽世界网友的同事都是在用中国移动的2G网络，数据传输只能靠GPRS。）
##2、Ubuntu下配置深度学习框架
2016年ALphaGo与李世乭的人机大战掀起了机器学习、深度学习的热潮，我于17年10月开始学习开始学习Python（一门在容易上手的编程语言，在机器学习领域拥有较高的占有率），18年初加入一个深度学习的QQ小组。深度学习对于电脑性能有着更高的要求，老旧的笔记本电脑已经不能胜任这一需求，在和妻子商议后，花费了￥7000+在京东上购买配件自己组装了台式电脑。在安装Win7的过程中总是不能正确地识别驱动键盘，于是在同事的推荐下安装了Win10。

根据组长安排的学习进度，我们在3月份集中两个星期的时间学习Linux操作系统，要求完成深度学习框架的配置。当时推荐的学习材料除了网易的公开课之外还有《鸟哥的Linux私房菜基础学习篇-第三版》。我下载ubuntu-16.04.4-desktop-amd64的版本，安装在台式机单独的一块硬盘中，经过多次的尝试成功地配置好GPU版本的tensorflow框架。但是自5月2日运行完“猫狗大战”的例子之后，没有继续投入时间和精力在深度学习方面。
##3、CentOS的学习
深度学习告一段落，但是Sophie组长推荐的电子书却给我留下了深刻的印象。之后我曾在IPad上断续的阅读该书的部分章节，并于18年6月购买了《鸟哥的Linux私房菜：服务器架设篇（第三版）》纸质书。学习《服务器架设》的过程中发现基础篇已经更新到第四版了，与第三版不同，基础篇的第四版已经改用CentOS7进行讲解。
2018年12月下载CentOS-7-x86_64-DVD-1810版本后，前后数次在ASUS笔记本电脑以及新台式机上安装CentOS7，并进行了一些操作练习。19年1月购买《基础学习篇（第四版）》纸质书，学习的同时开始做笔记记录一些要点。目前已学完前面九章，最后面的四个章章也粗略的看过一遍。
###实践及收获
参照教材第三章内容在ASUS笔记本电脑上安装了Linux和Win7双系统，实现了多重开机安装流程与管理（grub2）。几经周折安装好了shadowsocks-qt5客户端，对于Linux下的软件安装机制有了一定得认识。Linux水平自评——（初级）： 了解系统的启动流程及开机中的引导 ；基本了解Linux的文件系统及目录架构；知道Linux下文件及用户的权限；会在Linux对磁盘进行分区，格式化，挂载以及设置启动挂载。
##4、一些思考
18年12月重新启动CentOS的学习后，自己的思路不是很清晰，目标也不够明确。
对于笔记本、台式机以及硬盘资源的规划一直犹豫不决。一方面想一次到位，把Linux打造成自己顺手的工具；另一方面又舍弃不了用顺手了的Windows。安装、格式化、再次安装，来来回回折腾了几次，浪费了大量的时间。最后才定下来在旧笔记本上安装双系统，用于CentOS学习过程中的实践以及移动办公；台式机保持Win10主用操作系统，用于处理日常的其他事项。
Linux再经典、再强大，充其量也只是个操作系统，是一个工具。虽说“磨刀不误砍柴工”，但是一定要对自己所要砍的“柴”有清醒的认识，不能盲目地在工具上投入过多的经历，在耽于磨刀的过程中失去方向，还用所谓的“不误”来为自己的踌躇不前辩解。
